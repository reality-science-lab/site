## 1. グローバル仕様

### 1.1 ブレイクポイント

| 名称 | 幅 | 想定デバイス |
|---|---|---|
| mobile | 〜 767px | スマートフォン縦持ち |
| tablet | 768px〜1023px | タブレット・スマートフォン横持ち |
| desktop | 1024px〜 | PC・ラップトップ |

tablet はほぼ desktop レイアウトに準ずる（主な分岐は mobile/desktop の2系統）。  
デザインの基準幅（design viewport）：desktop = **1280px**、max-content-width = **1120px**。

### 1.2 グローバルナビゲーション — 原則「なし」

**判断根拠:** design.md §6.1「スクロールは次の素材を取り寄せる感覚で設計する」に基づき、従来型のナビバーは設置しない。代わりに以下を採用する。

- **§章番号マーカー（左マージン固定）** が現在地ナビゲーションを兼ねる（詳細は §1.3）
- **スクロール進行バー** がページ全体の進捗を示す
- ページ内に外部リンクが必要な箇所（フッター、プロジェクト詳細など）は各セクション内でインラインリンクとして処理する

**例外:** §09「参画」CTAへの固定リンクを、desktop のみ右下に小さく浮かせる（詳細は §2.09）。

**禁止:** ハンバーガーメニュー、ドロップダウン、スティッキーナビバーはすべて設置しない。「大学 LP らしさ」への最短距離になるため。

### 1.3 §章番号マーカーの固定表示

**位置:** デスクトップでは viewport 左端から 32px の固定位置（position: fixed, left: 32px）。  
**表示内容:** 現在スクロール位置のセクションに対応する `§ 0X` をモノスペースで表示。  
**フォント:** IBM Plex Mono / Regular / 11px / letter-spacing: +40  
**色:** `#8C8B85`（Structural Gray）— 普段は引いて存在し、主張しない  
**更新タイミング:** Intersection Observer でセクションの閾値（top が viewport 上端から 20% 通過）を検知して切り替え。トランジションは opacity 0→1、200ms、ease-out。

```
[固定位置 left:32px]
│
│  §03
│
│  （現在スクロール中のセクション番号）
```

**mobile:** 非表示。代わりにスクロール進行バーのみで現在地を示す。

### 1.4 スクロール進行マーカー

デスクトップ・モバイル共通。  
- viewport 右端から 0px、高さ = viewport 高さ の **1px 幅の縦線**（`#E8E6DF`）
- その上に、スクロール進捗に連動した **`#D6422E`（校正の赤）の進行バー** を重ねる
- 進行バーは `scroll-timeline` または `scrollY / documentHeight` で計算した `height%` で実装
- この赤は「スクロール=編集の進行」（design.md §5.2-2）のモーション表現であり、ページ内で常時見えている赤の唯一の例外として許容する

### 1.5 言語切替（JA/EN）

**方針:** LP v1 は日本語のみで公開。英語は §10 フッターに「English version coming soon」を1行添えるのみ。  
**理由:** 
- 言語切替 UI は「あなたは誰ですか」分岐と同じ種類の入力を訪問者に強いる（design.md §6.3「分岐ではなく読みながら立ち上がる」原則に抵触）
- コピーの質が言葉主体の設計の核であり（design.md §6.4）、翻訳品質が確保されるまでは露出しない
- 英語版が整った時点で `/en/` サブパスとして追加し、hreflang で接続する

### 1.6 フォントロード戦略

1. System font（游ゴシック体）は CSS font-family stack に先行定義、FOIT/FOUT を防ぐ
2. IBM Plex Mono（数字・章番号・キャプション用）は `<link rel="preload">` で優先ロード
3. A1 ゴシック StdN / 筑紫A見出ミン はサブセット化（使用文字のみ）して遅延ロード
4. Söhne は可能なら Fontsource 経由で self-hosting（GDPR・パフォーマンス両面から）

### 1.7 カラートークン（CSS カスタムプロパティ）

```css
:root {
  --color-paper:     #FAFAF7;  /* Paper White — 背景ベース */
  --color-ink:       #0F0F0E;  /* Ink Black — 本文 */
  --color-red:       #D6422E;  /* Editor's Red — 編集の一色、1スクリーン1点 */
  --color-gray-str:  #8C8B85;  /* Structural Gray — 罫線・キャプション */
  --color-gray-mrg:  #E8E6DF;  /* Margin Gray — セクション分割 */
  --color-blue:      #1F3A8A;  /* Manuscript Blue — リンク専用 */
}
```

#### インタラクション

- **スクロール登場（design.md §5.2-2）:** セクションが viewport に入ったとき、本文テキストが「マークアップが入るように」現れる。具体的には opacity: 0 → 1、transform: translateY(8px) → 0、400ms、cubic-bezier(0.2,0,0,1)。「下からスライドイン」ではなく「プリントされるように」現れる感覚を出すため translateY は最小値に抑える。
- 同時に2要素以上を動かさない。見出しが先に現れ（0ms delay）、本文が後（120ms delay）。

- 右半分の余白は意図的。「埋めない、加える」原則の最初の体現
- 「自由な時代」というキーワードには下線（`border-bottom: 1px solid #0F0F0E`）を一本引き、編集記号として機能させる（design.md §2.2 アンダーライン語彙）

## 4. レスポンシブ方針

### 4.1 §03 マトリクス図のモバイル対応

**問題:** 2列 × 4行のマトリクスは、モバイル画面幅（375〜430px）では横軸ラベルと縦軸ラベルが両立できない。

**解決策:** 横スクロール + ピン固定縦軸

```
[モバイルでの実装方針]

コンテナ: overflow-x: scroll、-webkit-overflow-scrolling: touch
縦軸ラベル（文化/創発/境界/自己）: position: sticky; left: 0 で左端に固定
横軸ラベル（内向き/外向き）: 通常フロー（スクロールで消える）

最小セル幅: 160px × 2列 = 320px（375px画面でも収まる）
縦軸ラベル幅: 64px（IBM Plex Mono / 10px / 縦書き）

スクロールインジケーター: 右端に gradient fade + 「← →」で示す
```

**代替案（コスト低）:** マトリクスを縦型リストに変換し、各行で「スケール → [内向き: ×] [外向き: ○]」と表示。ただし「思想の身体図」という性質が損なわれるため、横スクロール案を推奨する。

### 4.2 §08 学長メッセージ引用ブロックのモバイル組版

**問題:** 引用テキストが22px + line-height 1.9 でのデスクトップ設計は、モバイルで1行に入りきらない。

**方針:**
- モバイルでのフォントサイズ: 17px（デスクトップ22pxから縮小）
- line-height: 1.9 は維持（可読性のため）
- 引用ブロックの幅: 100%（左右パディング 24px）
- 縦罫（3px / #D6422E）は維持する（設計意図の核心）
- 1行あたりの文字数: 約20〜24文字（モバイルで自然な折り返し）

**組版上の注意:**
- 「あなたの色で、あなたのやり方で、あなたの可能性で——世界を満たそう。」の最終文は改行をコントロールして1フレーズが途中で切れないように `<span class="nobr">` または `white-space: nowrap` で保護する

### 4.3 §09 応募フォームのモバイルUX

| 要素 | デスクトップ | モバイル |
|---|---|---|
| フォームフィールド | 下線スタイル（border-bottom のみ） | 同左（変更なし） |
| ラベル | フィールド上部に配置 | 同左 |
| テキストエリア | 高さ 120px | 高さ 100px（ソフトキーボード考慮） |
| 送信ボタン | 幅 260px、中央揃え | width: 100%（フル幅）|
| 固定CTA | 右下 32px | 非表示（フォームが同一ページにあるため） |

**ソフトキーボード対応:**
- テキストエリア focus 時に `scroll-margin-top: 120px` を設定し、キーボード展開後もフィールドが見える位置を確保
- iOS Safari での `viewport` 問題を `dvh` 単位で回避

## 5. 推奨技術スタック

### 5.1 フロントエンド

**推奨: Astro v5**

| 観点 | 理由 |
|---|---|
| コンテンツ主体 | Astro はコンテンツファーストのSSGフレームワーク。JSを最小化しHTML+CSSを最大化するアーキテクチャが、「言葉が主役」の設計思想と一致する |
| パフォーマンス | デフォルトで JS ゼロ出力。LP として必要な Core Web Vitals（LCP < 2.5s）を達成しやすい |
| View Transitions | Astro 内蔵の View Transitions API ラッパーで、LP → プロジェクト詳細ページのクロスフェードが実装しやすい |
| 将来の拡張性 | コンテンツ層を Markdown/MDX に分離できる。将来の多言語化にも対応可能 |

**不採用の候補と理由:**
- Next.js: App Router のオーバーヘッドがこのシンプルな LP には過剰。React の hydration コストがパフォーマンスに不利
- SvelteKit: 優秀だが、Studio Saitama のエコシステムとの親和性・学習コストを考慮

### 5.2 アニメーション実装手段

**推奨: CSS アニメーション + Intersection Observer（JS 最小）**

design.md §5.2 で定義した4つの仕掛けの実装方針:

| 仕掛け | 実装手段 | 詳細 |
|---|---|---|
| Hero テキスト差し替え | JavaScript + CSS transition | `setInterval` で語を切り替え、取り消し線の width アニメーションは CSS transition で実装。Framer Motion / GSAP 不要 |
| スクロール=編集の進行 | Intersection Observer + CSS | セクション検知は Intersection Observer、登場アニメーションは CSS `@keyframes`。`animation-play-state: paused → running` で制御 |
| ホバー=マトリクス強調 | CSS `:hover` + カスタムプロパティ | JS なしで実装可能。`--active-row` `--active-col` などのカスタムプロパティをセルに設定し、親要素の `:has()` セレクタで他セルの opacity を制御 |
| クリック=diff 展開 | JavaScript（最小） | `details/summary` 要素ベースで実装し、CSS でスタイリング。JS は `<details>` の open 属性検知のみ |

**不採用の手段と理由:**
- Framer Motion: React 専用。Astro との組み合わせは可能だが、このLP の要件に対してオーバースペック
- GSAP: 機能は申し分ないが、ライセンス（商用は要確認）と bundle size。CSS + Intersection Observer で代替できる範囲に留める
- Lottie: design.md §5.3 で明示的に禁止

**View Transitions API（ページ遷移）:**
- LP → プロジェクト詳細ページの遷移に使用
- Astro の `<ViewTransitions />` コンポーネントで実装
- フォールバック: 対応ブラウザ外では通常のページ遷移

### 5.3 フォーム

**推奨: Tally（埋め込み型）+ カスタムCSS でスタイル上書き**

| 観点 | 理由 |
|---|---|
| 実装速度 | コーディング不要でフォーム構築。事務局が独自に管理可能 |
| スタイル | Tally の透過埋め込みモードを使い、CSS でフォームフィールドのデザインを完全にカスタム |
| データ | Notion / Google Sheets に自動連携可能 |
| コスト | 月 $0（Free プラン）でLP規模の利用は十分 |

**代替案:**
- Google Forms 埋め込み: スタイルのカスタマイズが困難。design.md のトーンを毀損するリスクがある
- 自前実装（Astro + Resend API）: 最高の体験品質だが、実装・保守コストがかかる。v2 以降での検討推奨

### 5.4 デプロイ・ホスティング

**推奨: Cloudflare Pages**
- 静的サイト（Astro SSG）のホスティングとして最速クラス
- DHU ドメインに CNAME 設定するだけで稼働
- Edge Network での配信でグローバルレイテンシを最小化

## 6. アクセシビリティ

### 6.1 校正の赤 `#D6422E` × 白 `#FAFAF7` のコントラスト比検証

**計算結果:**

相対輝度（WCAG 2.1 定義に基づく計算）:
- `#D6422E`: R=214, G=66, B=46 → 相対輝度 L ≈ 0.1893
- `#FAFAF7`: R=250, G=250, B=247 → 相対輝度 L ≈ 0.9557

コントラスト比 = (0.9557 + 0.05) / (0.1893 + 0.05) = **約 4.20:1**

| WCAG 基準 | 要求値 | 判定 |
|---|---|---|
| AA — 通常テキスト（< 18pt） | 4.5:1 | **不合格（4.20:1）** |
| AA — 大テキスト（≥ 18pt / Bold 14pt以上） | 3:1 | **合格** |
| AAA — 通常テキスト | 7:1 | 不合格 |
| AAA — 大テキスト | 4.5:1 | 不合格 |

**対応方針（重要）:**

校正の赤はテキストカラーとしては使用しない。design.md の設計意図通り、赤は以下の用途に限定する：

1. **取り消し線**（Hero テキスト差し替え）: テキスト本体は `#0F0F0E` のまま、取り消し線のみ赤。コントラスト問題は発生しない
2. **引用ブロックの縦罫**（§08）: 装飾的要素。WCAG の「コントラスト比の要件は非テキストコントラストを除く」規則により、罫線・装飾は 3:1 でよい → 合格
3. **アンダーライン**（§09 見出し下線）: 見出しテキスト自体は `#0F0F0E`。下線が装飾的であれば非テキストコントラスト基準（3:1）を適用 → 合格
4. **スクロール進行バー**（グローバル）: UI コンポーネント / 非テキスト → 3:1 基準 → 合格
5. **diff 展開の取り消し線テキスト**（§02）: 取り消し線付きの旧テキストは装飾的表現であり、同時に新テキスト（`#0F0F0E`）が隣に提示される。AAの観点では「情報の代替手段がある」として許容される

**禁止:** `#D6422E` を通常文字色として小サイズテキストに使用することは WCAG AA 違反のため絶対に行わない。

### 6.2 Hero テキスト差し替えアニメの `prefers-reduced-motion` 対応

```css
@media (prefers-reduced-motion: reduce) {
  /* テキスト差し替えアニメーションを完全停止 */
  .hero-word-swap {
    animation: none;
    transition: none;
  }
  /* 最初のワード「現実」を固定表示 */
  .hero-word-swap .word:not(:first-child) {
    display: none;
  }
  /* スクロール登場アニメーション無効化 */
  .scroll-reveal {
    animation: none;
    opacity: 1;
    transform: none;
  }
}
```

JavaScript 側でも `window.matchMedia('(prefers-reduced-motion: reduce)')` を検知し、`setInterval` の差し替えループを停止する。

### 6.4 スクリーンリーダーでの §章番号読み上げ

**問題:** 「§ 01」「§ 02」という表示は、スクリーンリーダーが「セクション 01」と読むか「§ 0 1」と読むか不定。

**実装方針:**

```html
<!-- 固定位置の §マーカー（視覚専用） -->
<div class="section-marker" aria-hidden="true">§ 03</div>

<!-- セクション本体の aria-label -->
<section 
  id="section-03"
  aria-labelledby="section-03-heading"
>
  <!-- セクション内のビジュアル章番号 -->
  <span class="section-number" aria-hidden="true">§ 03</span>
  
  <!-- スクリーンリーダー向けの意味的見出し -->
  <h2 id="section-03-heading">
    <span class="sr-only">セクション3: </span>
    現実編集の4スケール
  </h2>
</section>
```

`.sr-only` クラスで視覚的に非表示にしつつスクリーンリーダーには読ませる標準パターンを使用。

**§マーカーの固定位置表示（`position: fixed`）はすべて `aria-hidden="true"`** を付与し、スクリーンリーダーが読み上げるドキュメント構造から除外する。

### 6.5 フォーカスインジケーター

すべてのインタラクティブ要素に明確なフォーカスインジケーターを設定する。

```css
:focus-visible {
  outline: 2px solid #0F0F0E;
  outline-offset: 3px;
}

/* 暗い背景（フッター）では反転 */
.footer :focus-visible {
  outline-color: #FAFAF7;
}
```

ブラウザデフォルトの `:focus` リセット（`outline: none`）は絶対に行わない。
